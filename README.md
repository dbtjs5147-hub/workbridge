# WorkBridge

> 개발을 모르는 초보도, 아이디어만 있으면 **AI가 PRD(요구사항 정의서)로 정리**해 주고
> 검증된 개발자에게 **입찰**받아 **단계별 안전결제(에스크로)**로 거래하는 개발 외주 매칭 플랫폼 (MVP)

의뢰인의 모호한 아이디어를 **AI가 기능·일정·마일스톤·검수 기준으로 구조화**하고,
**비용은 AI가 아니라 검증된 개발자가 직접 입찰**(밀봉 — 입찰가는 의뢰인만 열람)해
**단계별 안전결제(에스크로)**로 거래합니다. "프로젝트를 어떻게 정의할지조차 막막한 초보"를 위한,
위시켓 같은 기존 마켓의 **앞단(pre-스펙)** 포지셔닝입니다.

> ⚠️ 이 프로젝트는 **데모(시연)용 MVP**입니다. 실제 결제·송금은 일어나지 않으며,
> 결제는 PG sandbox(테스트) 방식, 전자서명은 체크박스 동의 방식으로 구현되어 있습니다.

## 🚀 빠른 시작 (처음 한 번만)

> 필요한 것: [Node.js](https://nodejs.org) 18 이상 (현재 v24에서 동작 확인), **PostgreSQL 연결 주소**
> (무료 [Supabase](https://supabase.com) 추천 — 만드는 법은 [DEPLOY.md](./DEPLOY.md) 1단계 참고)

```bash
# 1) 프로젝트 폴더에서 패키지 설치
npm install

# 2) .env 파일을 만들고 DATABASE_URL 에 PostgreSQL 주소 입력
#    (.env.example 를 복사해서 .env 로)

# 3) 데이터베이스 표 생성 + 시연용 데이터 채우기 (한 번에)
npm run setup

# 4) 개발 서버 실행
npm run dev
```

이제 브라우저에서 **http://localhost:3000** 에 접속하면 됩니다.

> 📦 **인터넷에 공개(배포)하려면** → [DEPLOY.md](./DEPLOY.md) (Vercel + Supabase + Blob, 비개발자용 단계별 가이드)

> 💡 **메모리(RAM)가 부족한 컴퓨터라면** (`JavaScript heap out of memory` 오류가 뜨면):
> 개발 서버(`npm run dev`)는 메모리를 많이 씁니다. 다른 프로그램을 닫아 메모리를 확보하거나,
> 더 가벼운 **배포 모드**로 실행하세요:
> ```bash
> npm run build   # 한 번 빌드
> npm run start   # 가벼운 배포 서버 실행
> ```
> (단, 코드를 수정하며 개발할 때는 `npm run dev`를, 단순히 보기만 할 때는 `build`+`start`를 권장합니다.)

### 데모 계정 (비밀번호는 모두 `test1234`)

| 역할 | 이메일 |

## 🎬 5분 시연 시나리오

1. **의뢰인(`client@test.com`)으로 로그인**
2. 상단 **프로젝트 등록** → 아이디어를 자연어로 입력 (예: "동네 학원 수강 신청·관리 웹")
   → **✨ AI 분석 요청** 클릭 → AI 확인 질문에 답 → 기능·일정·마일스톤·검수 기준·**PRD**가 자동 생성됨
   (가격은 표시되지 않음 — 개발자 입찰로 결정) → 내용 확인·수정 후 **프로젝트 등록**
3. **프리랜서(`dev@test.com`)로 로그인** → 프로젝트 상세에서 **입찰**(제안가 직접 입력)
   → 🔒 입찰가는 의뢰인에게만 공개되는 **밀봉 입찰**. (※ `dev3@test.com`은 포트폴리오 미작성이라 입찰이 막혀, 프로필 게이트를 바로 체험 가능)
4. 다시 **의뢰인** → 기존 시드 프로젝트 **"병원 예약 웹앱 MVP"** → **지원자 관리**에서 입찰 비교(최저·평균·최고)
   → 지원자 **수락** (입찰가가 마일스톤 에스크로에 배분되고 계약 자동 생성)
5. **계약 페이지**에서 **전자 서명**
6. **프리랜서**로 로그인 → 같은 계약에서 **전자 서명** (양측 서명 완료)
7. 다시 **의뢰인** → 1차 마일스톤 **에스크로 결제(sandbox)**
8. **프리랜서** → **납품 요청**
9. **의뢰인** → **검수 승인** (수수료 10% 차감 후 정산, 다음 마일스톤으로 진행)
10. 모든 마일스톤 완료 → 프로젝트 **완료** → 양측 **상호 리뷰** 작성
11. **공개 프로필**에서 평균 별점·리뷰·주요 평가 태그 확인

> 이미 **"AI 문서요약 SaaS"**(작업 중)와 **"맛집 커뮤니티 앱"**(완료·리뷰 있음) 프로젝트가
> 시드 데이터로 준비되어 있어, 중간/완료 상태도 바로 둘러볼 수 있습니다.

## 🧩 기술 스택

- **프레임워크**: Next.js 15 (App Router) — 프론트엔드 + 백엔드 API를 한 앱에서 처리
- **언어**: TypeScript
- **스타일**: Tailwind CSS
- **데이터베이스**: PostgreSQL + Prisma ORM (로컬·운영 모두, Supabase 권장)
- **인증**: JWT (httpOnly 쿠키) + bcrypt 비밀번호 해싱
- **AI 분석**: mock 엔진 내장 (키 없이 동작) / Anthropic·OpenAI 키 입력 시 실제 LLM으로 자동 전환
- **파일 저장**: 로컬은 `public/uploads`, 운영은 Vercel Blob(`BLOB_READ_WRITE_TOKEN` 있으면 자동 전환)
- **결제**: PG sandbox(mock) — 멱등 처리, 실제 토스/포트원 연동 확장 가능
- **배포**: Vercel (GitHub `main` 푸시 시 자동 배포)

## 🔑 환경 변수 (`.env`)

`.env.example`를 복사해 `.env`로 만들면 됩니다. **`DATABASE_URL` 외에는 비어 있어도 동작**합니다.

| 변수 | 설명 | 비워두면 |
|------|------|----------|
| `DATABASE_URL` | PostgreSQL 연결 주소 (Supabase) | ⚠️ **필수** (없으면 DB 동작 안 함) |
| `JWT_SECRET` | 로그인 토큰 서명 키 | (기본값 있음, 운영 시 꼭 변경) |
| `BLOB_READ_WRITE_TOKEN` | Vercel Blob 파일 저장소 토큰 | 로컬 폴더 저장(배포 시 휘발) |
| `AI_PROVIDER` | `mock`/`anthropic`/`openai` | mock 분석 엔진 동작 |
| `ANTHROPIC_API_KEY` / `OPENAI_API_KEY` | 실제 AI 키 | mock 사용 |
| `GOOGLE_*` / `KAKAO_*` | 소셜 로그인 | 데모 소셜 로그인 동작 |
| `PG_PROVIDER` / `TOSS_*` | 결제 | mock 결제 동작 |

실제 AI를 쓰려면 `.env`에서 `AI_PROVIDER="anthropic"`로 바꾸고 `ANTHROPIC_API_KEY`만 넣으면 됩니다.

## ✅ 구현된 핵심 기능

- 이메일/소셜(데모) 회원가입·로그인, 역할 선택, 역할별 프로필
- **AI 프로젝트 분석** (기능 분해·일정·기술스택·마일스톤·검수기준·리스크·다단계 확인질문·계약범위 초안)
- **AI PRD 문서 생성** — 아이디어 → AI 분석 → 의뢰인이 확인 질문에 답변 → 완성된 PRD(요구사항 정의서) 자동 생성 → 계약된 개발자에게 전달 (의뢰인↔개발자 이해 차이 해소)
- **무료 체험(비로그인)** — 가입 없이 `/try`에서 AI 분석·PRD를 받아보고, 가입하면 그 결과로 바로 프로젝트 등록 (유입 깔때기)
- **변경 요청** — 계약 후 추가/수정 요청 시 AI가 "기존 범위 포함 여부" 참고 의견 제시, 수락하면 추가 마일스톤으로 안전하게 처리
- 프로젝트 등록·탐색(검색/필터/정렬)·상세·유사 프로젝트 추천·북마크
- 프로젝트 지원(**입찰**), 지원자 비교, 수락/거절, **자동 거절**, 인앱 알림
- **밀봉 입찰 가격 모델** — 플랫폼/AI는 금액을 정하지 않음. 검증된 개발자가 **제안가를 직접 입찰**하고, 입찰가는 **의뢰인에게만 공개**(프리랜서끼리 비공개). 의뢰인은 입찰 요약(최저·평균·최고)으로 비교, 수락 시 입찰가가 마일스톤 에스크로에 자동 배분
- **프리랜서 입찰 자격 검증** — "검증된 개발자" 신뢰를 뒷받침하기 위해, 빈 프로필로는 입찰 불가. 최소 **기술 스택 + 포트폴리오**를 갖춰야 입찰 가능 (화면·서버 양쪽에서 검증)
- **계약서 자동 생성 → 양측 전자 서명**
- **마일스톤 에스크로**: PG sandbox 결제 → 납품 요청 → 검수 승인/수정 요청 → **수수료 10% 차감 정산**
- 프로젝트 완료 → 상호 리뷰(별점·태그) → 공개 프로필 평판 반영
- **실시간 채팅** — 의뢰인↔프리랜서 1:1 메시지 (프로젝트 상세/지원자 목록에서 시작, 헤더에 안읽음 배지)
- **파일 업로드** — 프로필 사진, 포트폴리오(이미지·PDF), 납품물 첨부 (이미지는 썸네일, PDF는 파일 링크). 데모는 로컬 저장(`/public/uploads`), 운영 시 클라우드 스토리지로 교체
- 서버 측 상태 전이 검증, 소유자 기반 권한 제어, 중복 방지 제약
